# Техническое задание (ТЗ) и пошаговая инструкция по разработке новостного портала «МатрёЖка» на WordPress

## 1. Общая концепция и архитектура проекта

**Цель:** создать отдельный, полностью функциональный новостной сайт на WordPress, который будет служить «движком» для проекта «МатрёЖка» и единым источником данных (бэкендом) для мобильного приложения.

**Архитектурное решение:**

1. Основной сайт-визитка `matrezhka.ru` не участвует в данной разработке. Он остаётся на текущем конструкторе (LPmotor) и служит для привлечения новых авторов (SEO-воронка) и презентации проекта.
2. Данное ТЗ описывает создание нового, полностью независимого сайта на поддомене `news.matrezhka.ru`. Этот сайт отвечает за публикацию новостей, модерацию и предоставление данных через REST API.

### 1.1. Уточнения заказчика (приоритетные)

1. `matrezhka.ru` остаётся как есть (лендинг/SEO-воронка), не переносится на WordPress в рамках этой задачи.
2. Новый `news.matrezhka.ru` должен визуально и структурно повторять текущую новостную страницу на основном домене (страницу вида `matrezhka.ru/news`):
   - идентичная шапка,
   - идентичный подвал,
   - аналогичная концепция новостной ленты.
3. Все разделы и ссылки в шапке/подвале должны быть сохранены в эквивалентном виде для последующей перелинковки домен <-> поддомен.
4. ТЗ — это рамка, а не ограничение. Реализация должна быть выбрана с упором на результат, расширяемость и дальнейшее развитие проекта.
5. Личные кабинеты/учётные записи редакционного процесса должны быть предусмотрены (минимум роли и права для авторов/модерации).

## 2. Этап 1: Подготовка хостинга и установка WordPress

### Шаг 1.1. Создание поддомена

- В панели управления хостингом, где расположен основной сайт `matrezhka.ru`, создать поддомен `news.matrezhka.ru`.
- Папка для поддомена может быть `public_html/news/` или аналогичная.

### Шаг 1.2. Установка WordPress и SSL

- Для поддомена `news.matrezhka.ru` выпустить SSL-сертификат (обычно бесплатный Let's Encrypt через панель хостинга).
- Установить последнюю версию WordPress (русский язык).
- После установки войти в админку: `https://news.matrezhka.ru/wp-admin`.

### Шаг 1.3. Базовая настройка WordPress

- `Настройки -> Постоянные ссылки`: выбрать вариант **«Название записи»** (`https://news.matrezhka.ru/sample-post/`).
- Удалить стандартные неиспользуемые плагины (`Akismet`, `Hello Dolly`) и тестовый контент (запись «Привет, мир!», страница «Пример страницы»).

## 3. Этап 2: Разработка кастомной темы (без GeneratePress)

### Шаг 2.1. Создание кастомной темы

- Создать новую тему, например `matrezhka-news`, в каталоге `wp-content/themes/matrezhka-news`.
- Минимальный набор файлов:
  - `style.css`
  - `functions.php`
  - `index.php`
  - `header.php`
  - `footer.php`
  - `home.php` (лента новостей)
  - `single.php` (детальная новость)
  - `archive.php` (архивы/рубрики)
  - `page.php` (статические страницы)
- Активировать тему в `Внешний вид -> Темы`.

### Шаг 2.2. Базовая архитектура темы

- В `functions.php` зарегистрировать:
  - меню (`register_nav_menus`)
  - области виджетов (`register_sidebar`)
  - поддержку `title-tag`, `post-thumbnails`, `html5`.
- Подключить стили и скрипты через `wp_enqueue_scripts`.
- Реализовать адаптивный layout:
  - контейнер контента (например, `max-width: 1200px`)
  - сетка/список для ленты новостей
  - блок метаданных (автор, дата, рубрика, город).
- Шапка и подвал должны быть собраны как эквивалент текущих блоков с `matrezhka.ru/news`:
  - те же ключевые пункты меню,
  - те же разделы/ссылки,
  - та же логика навигации.
- В подвале обязательно сохранить ссылку «О проекте» на `https://matrezhka.ru` и копирайт.

## 4. Этап 3: Установка и настройка ключевых плагинов

### Шаг 3.1. Список плагинов для установки

Через `Плагины -> Добавить новый` найти, установить и активировать:

- `Advanced Custom Fields (ACF)` — для дополнительных полей записей.
- `User Role Editor` — для создания/управления ролями пользователей.
- `JWT Authentication for WP REST API` — для авторизации в мобильном приложении.
- `WP ULike` — для функционала лайков.
- `Ad Inserter` — для рекламных блоков.

`Custom Post Type UI` в этом проекте **не обязателен**.

## 5. Этап 4: Создание структуры контента

### Шаг 5.1. Создание рубрик (категории)

`Записи -> Рубрики`: создать стандартные рубрики (slug на латинице):

- Новости (`novosti`)
- Интервью (`intervyu`)
- Школа (`shkola`)
- Кино и игры (`kino-igry`)
- Спорт (`sport`)
- Творчество (`tvorchestvo`)
- Путешествия (`puteshestviya`)
- Хобби (`hobbi`)

### Шаг 5.2. Создание таксономии «Города»

Рекомендуемый вариант: зарегистрировать таксономию `city` кодом в `functions.php` темы или в `mu-plugin` (предпочтительно для стабильности).

```php
add_action('init', function () {
    register_taxonomy('city', ['post'], [
        'labels' => [
            'name' => 'Города',
            'singular_name' => 'Город',
        ],
        'public' => true,
        'show_ui' => true,
        'show_in_rest' => true,
        'rest_base' => 'city',
        'hierarchical' => false,
        'rewrite' => ['slug' => 'city'],
    ]);
});
```

Критичные параметры:

- `hierarchical = false` (важно для производительности при 1130 городах).
- `show_in_rest = true` (обязательно для мобильного приложения).
- `rest_base = city`.

Альтернатива: если используется **ACF Pro** с UI для таксономий, таксономию можно создать через ACF, но параметры должны быть теми же.

### Шаг 5.3. Массовый импорт городов (1130 штук)

- Для импорта использовать плагин типа `Bulk Term Generator` или `Quick Term Creator`.
- Импортировать в таксономию `city` список городов из CSV (колонки `name`, `slug`).

## 6. Этап 5: Настройка дополнительных полей (ACF)

### Шаг 6.1. Создание группы полей

- `ACF -> Группы полей -> Добавить новую`
- Заголовок группы: **Параметры новости**
- `Настройки -> Местоположение`: `Тип записи = Запись`

### Шаг 6.2. Добавление полей

Для каждого поля включить опцию **«Показывать в REST API»**.

| Название поля | Тип поля | Обязательное | Настройки |
|---|---|---|---|
| Город | Таксономия | Да | Таксономия: `city`. Тип: выпадающий список. Сохранять значение: `ID`. |
| Выбор редакции | Истина/Ложь | Нет | Текст метки: «Выбор редакции» |
| Срочная новость | Истина/Ложь | Нет | Текст метки: «Срочная новость» |
| Правки редактора | Текстовая область | Нет | Текст метки: «Правки редактору» |

После добавления полей нажать `Опубликовать`.

## 7. Этап 6: Настройка ролей и прав пользователей

### Шаг 7.1. Настройка ролей через User Role Editor

`Пользователи -> User Role Editor`: создать/настроить роли:

| Роль | Ключевые права (Capabilities) | Точка входа и интерфейс |
|---|---|---|
| Автор | `create_posts`, `edit_posts`, `upload_files`, `delete_posts` (только свои), без `publish_posts` | Админка WordPress (`/wp-admin`) и/или мобильное приложение |
| Модератор | Права Автора + `edit_others_posts`, `publish_posts`, `delete_posts`, `moderate_comments` | Только админка WordPress (`/wp-admin`) |
| Редактор | Права Модератора + `manage_categories`, `edit_theme_options` | Только админка WordPress (`/wp-admin`) |

## 8. Этап 7: Настройка REST API для мобильного приложения

### Шаг 8.1. Настройка JWT-авторизации с возвратом роли

- Убедиться, что плагин `JWT Authentication for WP REST API` установлен и активирован.
- Добавить в `functions.php` активной темы код:

```php
add_filter( 'jwt_auth_token_before_dispatch', function( $data, $user ) {
    $data['roles'] = $user->roles;
    return $data;
}, 10, 2 );
```

Этот фильтр добавляет массив `roles` в ответ при успешной авторизации.

### Шаг 8.2. Проверка доступности данных в API

- Проверить доступность таксономии `city`:
  `https://news.matrezhka.ru/wp-json/wp/v2/city`
- Создать тестовую запись, заполнить ACF-поля и опубликовать.
- Проверить доступность ACF в REST:
  `https://news.matrezhka.ru/wp-json/wp/v2/posts/ID_тестовой_записи`
- В JSON должен присутствовать объект `acf` со всеми созданными полями.

### Шаг 8.3. Кастомный эндпоинт для рекламы

Создать эндпоинт:

- `GET /wp-json/matrezhka/v1/ads`

Пример реализации в `functions.php`:

```php
add_action( 'rest_api_init', function () {
    register_rest_route( 'matrezhka/v1', '/ads', array(
        'methods'  => 'GET',
        'callback' => function ( $data ) {
            // Здесь должна быть логика получения списка активных рекламных блоков.
            // Пока возвращается тестовый массив.
            $ads = array(
                array( 'id' => 1, 'type' => 'banner', 'content' => '<img src="..." />', 'position' => 'header' )
            );
            return $ads;
        },
    ) );
} );
```
